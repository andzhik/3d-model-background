## Prompt 03 — Add the WebGL shell and safe fallback

- **Completion date:** 2026-07-14
- **Outcome:** Added a decorative React Three Fiber canvas between the permanent poster and semantic hero content. The new `SceneBackground` owns renderer lifecycle and fallback state, while `Experience` supplies a perspective camera configuration, hemisphere and ambient fill, a warm directional light, and a visibly faceted icosahedron placeholder.
- **Loading and fallback decisions:** The canvas begins fully transparent and reports readiness only after a frame with a live WebGL context and finite camera matrix has rendered. Readiness is deferred to the following animation frame so the browser can paint the completed scene before the 600 ms fade begins. WebGL capability failure, React render errors, and `webglcontextlost` all move the layer to a terminal hidden fallback state. The poster remains the hero's CSS background in loading, success, and failure states, preventing a blank startup frame.
- **Accessibility:** Both the scene wrapper and renderer are `aria-hidden`; the entire scene layer has pointer events disabled. The title, supporting copy, and CTA remain ordinary semantic HTML above the canvas and contrast scrim.
- **Automated protection:** Added reducer tests for first-frame readiness, failures from every lifecycle state, and a late readiness callback after failure. Added a component test confirming that an environment without WebGL retains the fallback state and mounts no canvas.
- **Verification:** `npm run typecheck`, `npm run lint`, `npm test`, `npm run build`, and `npm run format:check` passed. Vitest ran 2 test files with 8 passing tests. Vite emitted the production build successfully: 1.18 kB HTML (0.56 kB gzip), 3.29 kB CSS (1.39 kB gzip), 1,075.36 kB JavaScript (295.33 kB gzip), and the unchanged 2,058.75 kB poster asset.
- **Remaining limitations:** The faceted object and lighting are intentionally temporary and will be replaced by later scene prompts. Vite reports that the initial JavaScript chunk exceeds 500 kB; code splitting and measured optimization remain future work once the scene architecture stabilizes. A live browser smoke check was attempted, but the available in-app browser runtime failed during initialization (`Cannot redefine property: process`), so no successful WebGL screenshot or manual visual result is claimed for this step.

## Prompt 05 — Build the sky, sun, and clouds

- **Completion date:** 2026-07-14
- **Outcome:** Replaced the faceted placeholder with a procedural atmospheric scene containing a three-stop low-poly-reference sky gradient, an unlit warm sun disc near the horizon, and 12 polygonal clouds split across far and near depth layers. The supplied poster remains permanently visible beneath the translucent WebGL layer while later environment features are incomplete.
- **Composition decisions:** Kept the sun at world `x = 0` so the centered perspective camera projects it onto the central vertical axis at every target aspect ratio. Placed it below camera center at `y = -1.45` to establish the future valley horizon. The oversized sky plane covers the widest target frustum without relying on responsive camera changes reserved for Prompt 14.
- **Determinism and motion:** Cloud counts, positions, scales, and one of three faceted silhouettes come from the existing seeded random generator and named atmosphere configuration. Far clouds sit at `z = -10` and drift at `0.018` world units per second; near clouds sit at `z = -5` and drift at `0.038` world units per second. Positions are calculated from elapsed seconds rather than accumulated frame deltas, then wrapped into a fixed field. A `useSyncExternalStore` reduced-motion hook tracks `prefers-reduced-motion` and freezes both layers at their deterministic initial positions.
- **Automated protection:** Added tests for repeatable placement, configured range bounds, elapsed-time drift and wrapping, and the near layer's greater depth and speed. The suite now contains 4 test files with 15 passing tests.
- **Verification:** `npm run typecheck`, `npm run lint`, `npm test`, `npm run build`, and `npm run format:check` passed. Vite emitted 1.18 kB HTML (0.56 kB gzip), 4.17 kB CSS (1.67 kB gzip), 1,082.23 kB JavaScript (297.89 kB gzip), and the unchanged 2,058.75 kB poster asset. Browser viewport inspection was attempted, but the available in-app browser again failed during initialization (`Cannot redefine property: process`), so no rendered screenshot inspection is claimed. Central sun alignment is guaranteed by the fixed zero X position and centered camera projection, and is covered across aspect ratios by the camera/plane geometry.
- **Remaining limitations:** Mountains, lake, shore, and character geometry are intentionally absent, so the poster remains visually important beneath the procedural atmosphere. Live viewport screenshots remain pending a working browser runtime. The existing Vite large-chunk warning remains reserved for later measured optimization prompts.
